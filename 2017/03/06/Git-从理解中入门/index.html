Git-从理解中入门

发表于 2017-03-06 | 分类于 git

前言：本篇文章适合入门者，着重理解了每一步的原理，拒绝死记硬背，虽然以前看过许许多多的教程与文档，也更为详细更为全面，但相对却也过于臃肿，而本篇文章注重于个人学习过程中git的使用
（本文为个人瞎逼逼，如有纰漏，请尽情指出）

Git概念及四大块

Git是一种分布式版本管理系统（区别于SVN那种集中型版本管理系统），开发者可以将其他开发者的仓库fork到自己的账户下，fork下来的仓库与原仓库就是两个不同的仓库，可以随意编写改动。（集中型版本管理系统详见附录）

workspace

workspace（工作区）会自动记录每个改动过程，但不会被记入Git仓库的版本管理，比如，你在workspace添加了一个新文件又将其删除，则这个新文件的添加删除历史不会出现在版本管理中，如同从来没有出现过一样。

index

index（暂存区）是提交前的一个临时区域，可以让文件成为Git仓库的管理对象，就比如，你向index添加了一个新文件又将其删除，则这个新文件的添加删除历史都会出现在版本管理中。

local repository

local repository（本地仓库）可以将index中文件实际保存到仓库的历史记录中，通过历史记录可以在workspace中复原文件。

remote repository

remote repository（远程仓库）是代码托管的产物，实际上就是把我们的local repository托管到托管服务商（Github、Bitbucket、coding）的服务器上，便于实现代码共享。

基础操作（代码托管使用Github）

默认电脑已安装git

初始化远程仓库

在Github上新建一个新仓库（如果本地项目已存在README可不初始化README），复制好仓库SSH地址。

初始化本地仓库

运行控制台使用cd命令进入项目根目录，执行命令
$ git init
为这个项目新建了一个仓库，项目目录名要与Github上远程仓库一致。

连接本地仓库与远程仓库

$ git remote add origin /the remote repository SSH/
origin表远程，the remote repository SSH 为第一步复制好的远程仓库的SSH地址，也可以使用http地址。

拉取远程仓库与本地仓库合并

（如果远程仓库没有初始化README一般可以跳过此步骤）
执行命令
$ git pull origin master
该命令主要用来解决远程仓库与本地仓库发生冲突时使用，origin表远程，master表master分支。

添加文件到index

$ git add -A
如果本次改动并没有添加新文件，可跳过此步骤。

提交改动到本地仓库

（如果本次改动没有添加新文件，直接使用am即可）
$ git commit -am “ “
双引号里面是改动说明，不允许为空。
（正常步骤）
$ git commit -m ““
可以看出-am其实是-A-m的缩写。

推送改动到远程仓库

$ git push -u origin master
将改动推送到远程仓库master分支，之后的推送可以直接使用git push。

分支的操作

学完基本操作以后，我们知道Git不仅是个人用来开发学习使用，也可以应用在协同合作上，所以必须学会使用分支，即使是个人学习，Git推荐工作流程也是分支👉 编辑👉 提交👉 合并，因为可以减少在主分支上的错误操作。

创建分支

在当前分支上新建一个modify-README分支，并将modify-README设为当前分支。
$ git checkout -b modify-README
通过git branch命令我们也可以发现modify-README分支前有一个*号，表示为当前分支

在分支上提交

我们可以使用前一节提到的add和commit命令将改动提交到暂存区和仓库中。

合并

合并之前，我们应将当前分支修改为主分支（这里使用master作为主分支）
$ git checkout master
在主分支中将分支内容合并在主分支中
$ git merge modify-READE

清理无用分支

$ git branch modify-READE

附录

集中型版本管理系统

集中型将所有数据都集中存放在服务器中，有便于管理的优点，但一旦开发者无法连接服务器就无法正常进行工作，万一服务器出现问题，可能就再也无法看到源代码了。

git pull —rebase 和 git pull 的区别

git pull = git fetch + git merge
git pull —rebase = git pull + git rebase
git merge会将当前分支的提交保留，而git rebase会失去当前分支的提交记录。

git fetch

$ git fetch origin master
表现为取回origin master分支上的内容到本地的workspace。
